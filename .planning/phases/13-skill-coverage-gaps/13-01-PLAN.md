<objective>
Extend the skill dashboard to surface stale skills — those with no usage in the last 30 days — so Luis can identify and prune dead skills before they accumulate.

Purpose: Today the dashboard shows total skill count and contribution counts but gives no signal about which skills are actually being exercised. A 30-day-dark skill is a candidate for pruning from the master dir; this plan makes that candidate list observable.

Output:
- `GET /api/skills` response gains a `coverageGaps: string[]` field.
- Cookbooks node in the Flow canvas displays the gap count ("N stale (30d+)") alongside existing stats.
- Vitest coverage for the gap computation, covering fresh, stale, missing-state, and inaccessible-dir paths.

Scope guard: this plan ONLY adds coverageGaps. It does NOT modify totalSkills, contributedByHermes/Gwen, recentContributions, lastPruned, or lastUpdated logic.
</objective>

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem → API route | ~/.openclaw/skill-sync-state.json is a local file but `skill_usage` keys are skill directory names written by skill-sync.py — treat as data, not instruction |
| API route → client | coverageGaps strings are rendered in the Flow canvas subtitle/stats |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Tampering | skill-sync-state.json `skill_usage` dict | mitigate | Wrap `JSON.parse` in try/catch; on parse failure, treat as missing state (coverageGaps = full skill list). Never trust that values are valid timestamps — use `Number.isFinite(new Date(v).getTime())` guard before comparing. |
| T-13-02 | Denial of Service | SKILLS_PATH readdir | accept | Shallow `readdir` only (already enforced by existing code). No recursion. Skill dir count is bounded by master dir size (tens, not thousands). Existing catch block returns 0 on failure. |
| T-13-03 | Information Disclosure | coverageGaps list in HTTP response | accept | Skill directory names are not secret; they are already surfaced via totalSkills count and recentContributions.skill. No new PII exposed. |
| T-13-04 | Injection (XSS) | Cookbooks node rendering coverageGaps.length | mitigate | Only render `coverageGaps.length` (a number), NOT the string array, in the flow node. If later plans need to render names, rely on React's default text-node escaping (no raw HTML injection APIs). |
| T-13-05 | Elevation of Privilege | execSync/exec introduction | mitigate | Explicit prohibition in task action — this plan uses `fs/promises` only. Reference PROJECT.md constraint. |
</threat_model>
