**Step 5 -- Create src/app/api/time-series/route.ts:**

```typescript
import type { NextRequest } from 'next/server';
import { getDb } from '@/lib/db';
import { readFileSync } from 'node:fs';
import { SKILL_CONTRIBUTIONS_LOG, FAILURES_LOG } from '@/lib/constants';

export const dynamic = 'force-dynamic';

const VALID_METRICS = ['docs_ingested', 'memory_writes', 'recall_queries', 'collection_growth', 'skill_executions', 'skill_failures'] as const;
type Metric = typeof VALID_METRICS[number];

const VALID_WINDOWS = ['day', 'week', 'month'] as const;
type Window = typeof VALID_WINDOWS[number];

interface TimePoint { bucket: string; value: number; }
```

Helper `getWindowConfig(window: Window)` returns `{ since: string, bucketFormat: string }`:
- day: since = `datetime('now', '-1 day')`, bucketFormat = `'%H:00'`
- week: since = `datetime('now', '-7 days')`, bucketFormat = `'%Y-%m-%d'`
- month: since = `datetime('now', '-30 days')`, bucketFormat = `'%Y-%m-%d'`

The GET handler:
1. Parse metric and window from searchParams; validate against VALID_METRICS and VALID_WINDOWS;
   return 400 `{ error: 'Invalid metric' }` or `{ error: 'Invalid window' }` if invalid.
2. Branch on metric:

**SQL-based metrics (use getDb()):**

- `docs_ingested`:
  ```sql
  SELECT strftime(?, ingested_at) AS bucket, COUNT(*) AS value
  FROM ingest_meta WHERE ingested_at >= ?
  GROUP BY bucket ORDER BY bucket
  ```
  Pass bucketFormat and since as parameters.

- `memory_writes`:
  ```sql
  SELECT strftime(?, completed_at) AS bucket, SUM(insights_written) AS value
  FROM memory_consolidation_runs
  WHERE completed_at IS NOT NULL AND completed_at >= ?
  GROUP BY bucket ORDER BY bucket
  ```

- `recall_queries`:
  ```sql
  SELECT strftime(?, timestamp) AS bucket, COUNT(*) AS value
  FROM recall_log WHERE timestamp >= ?
  GROUP BY bucket ORDER BY bucket
  ```

- `collection_growth`:
  ```sql
  SELECT strftime(?, ingested_at) AS bucket, COUNT(DISTINCT file_path) AS value
  FROM ingest_meta WHERE ingested_at >= ?
  GROUP BY bucket ORDER BY bucket
  ```
  NOTE: ingest_meta has no "collection" column. This counts distinct file_path entries
  per time bucket -- effectively the ingest growth rate (new unique files per period).

**JSONL-based metrics (filesystem read):**

- `skill_executions`: Read SKILL_CONTRIBUTIONS_LOG via readFileSync. Wrap in try/catch --
  if file doesn't exist, return empty points array.
- `skill_failures`: Same approach but read FAILURES_LOG.

For JSONL parsing, create helper `parseJsonlBuckets(filePath: string, window: Window)`:
```typescript
function parseJsonlBuckets(filePath: string, window: Window): TimePoint[] {
  let raw: string;
  try { raw = readFileSync(filePath, 'utf-8'); } catch { return []; }
  const cfg = getJsWindowConfig(window);
  const sinceDate = cfg.since;
  const counts = new Map<string, number>();
  for (const line of raw.split('\n')) {
    if (!line.trim()) continue;
    try {
      const entry = JSON.parse(line);
      const ts = entry.timestamp || entry.ts;
      if (!ts) continue;
      const d = new Date(ts);
      if (d < sinceDate) continue;
      const bucket = cfg.bucketFn(d);
      counts.set(bucket, (counts.get(bucket) ?? 0) + 1);
    } catch { continue; }
  }
  return Array.from(counts.entries())
    .map(([bucket, value]) => ({ bucket, value }))
    .sort((a, b) => a.bucket.localeCompare(b.bucket));
}
```

`getJsWindowConfig(window)` returns:
- day: since = `new Date(Date.now() - 86400000)`, bucketFn = `(d) => d.toISOString().slice(11,13) + ':00'`
- week: since = `new Date(Date.now() - 7*86400000)`, bucketFn = `(d) => d.toISOString().slice(0,10)`
- month: since = `new Date(Date.now() - 30*86400000)`, bucketFn = `(d) => d.toISOString().slice(0,10)`

3. Return: `Response.json({ points, metric, window: windowParam, timestamp: new Date().toISOString() })`

```typescript
export type TimeSeriesMetric = 'docs_ingested' | 'memory_writes' | 'recall_queries' | 'collection_growth' | 'skill_executions' | 'skill_failures';
export type TimeSeriesWindow = 'day' | 'week' | 'month';

export function useTimeSeries(metric: TimeSeriesMetric, window: TimeSeriesWindow) {
  return useQuery({
    queryKey: ['time-series', metric, window],
    queryFn: () =>
      fetchJSON<{
        points: Array<{ bucket: string; value: number }>;
        metric: string;
        window: string;
        timestamp: string;
      }>(`/api/time-series?metric=${metric}&window=${window}`),
    refetchInterval: POLL_INTERVALS.knowledge, // 60s -- analytics don't need real-time
  });
}
```

This uses POLL_INTERVALS.knowledge (60000ms) as specified. The types TimeSeriesMetric
and TimeSeriesWindow are exported so Plan 02 components can import them for props.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> GET /api/time-series | Metric and window params from URL query string; untrusted input |
| client -> GET /api/recall | Query param q from URL; already validated but now also writes to recall_log |
| filesystem -> JSONL parse | JSONL files on disk could contain malformed JSON or unexpected fields |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-01 | Tampering | GET /api/time-series metric/window params | mitigate | Validate against allowlists (VALID_METRICS, VALID_WINDOWS); reject with 400 before any DB query |
| T-25-02 | Tampering | SQL injection via metric/window in time-series route | mitigate | metric and window are validated against string literal unions before use; SQL bucket format strings are hardcoded constants, never from user input; WHERE clauses use parameterized values |
| T-25-03 | Denial of Service | JSONL file read for skill_executions/skill_failures | accept | Single-user local tool; JSONL files are small (skill contributions); readFileSync is acceptable for local analytics; no amplification vector |
| T-25-04 | Information Disclosure | recall_log stores query text | accept | Single-user local tool; user's own queries; no PII beyond what user typed; no external access |
| T-25-05 | Denial of Service | recall_log unbounded growth | accept | Append-only log; single-user tool generates minimal queries; can add cleanup later if needed |
</threat_model>
