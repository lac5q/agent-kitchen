   Note: `constants.ts` is used client-side too. The `path` import will fail on client. Instead, use a conditional or inline the path join:
   ```typescript

   ```
   The DB singleton in `db.ts` will resolve this relative to `process.cwd()`.

2. Create `src/lib/db-schema.ts`:
   - Export `initSchema(db: Database.Database): void` function
   - Execute DDL for `messages` table with columns: id (INTEGER PRIMARY KEY), session_id (TEXT NOT NULL), project (TEXT NOT NULL), agent_id (TEXT NOT NULL), role (TEXT NOT NULL), content (TEXT NOT NULL), timestamp (TEXT NOT NULL), cwd (TEXT), git_branch (TEXT), request_id (TEXT), UNIQUE(session_id, request_id)
   - Execute DDL for `messages_fts` FTS5 virtual table using external content pointing at `messages`, columns: content, project UNINDEXED, timestamp UNINDEXED, agent_id UNINDEXED, tokenize='unicode61'
   - Execute DDL for AFTER INSERT trigger `messages_ai` on messages that syncs to messages_fts
   - Execute DDL for `ingest_meta` table: file_path (TEXT PRIMARY KEY), mtime_ms (INTEGER NOT NULL), file_size (INTEGER NOT NULL), row_count (INTEGER NOT NULL DEFAULT 0), ingested_at (TEXT NOT NULL)
   - Execute DDL for `meta` table: key (TEXT PRIMARY KEY), value (TEXT)

   - Module-level `_db` variable (null initially)
   - Export `getDb()`: if `_db` is null, resolve `SQLITE_DB_PATH` using `path.resolve(process.cwd(), SQLITE_DB_PATH)`, create directory with `fs.mkdirSync(path.dirname(resolved), { recursive: true })`, open Database, set `pragma journal_mode = WAL`, set `pragma synchronous = NORMAL`, call `initSchema(db)`, cache in `_db`
   - Export `closeDb()` for testing: close `_db` and set to null

4. Create test file `src/lib/__tests__/db.test.ts` with the behaviors listed above. Use a temporary directory for the test DB (override SQLITE_DB_PATH via env or pass path directly). Clean up temp DB in afterAll.
