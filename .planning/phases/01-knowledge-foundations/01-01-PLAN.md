<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| config file -> API route | `basePath` values in collections.config.json are read by the server to resolve filesystem paths |
| shell script -> cron | refresh-index.sh is executed by cron with user permissions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | T (Tampering) | collections.config.json basePath | accept | basePath values are hardcoded absolute paths to known directories, not user-supplied input. Server reads them at runtime. Single-user local app — no untrusted config writers. |
| T-01-02 | I (Info Disclosure) | basePath contains absolute home directory path | accept | Single-user local dashboard, no public exposure. Absolute path is required by the API's path resolution logic. |
| T-01-03 | T (Tampering) | refresh-index.sh modified to call external script | accept | The called script (gitnexus-index.sh) already exists and runs daily via cron. Adding it to the weekly script introduces no new attack surface. Both scripts are owned by the user. |
| T-01-04 | D (Denial of Service) | gitnexus-index.sh failure blocking qmd update | mitigate | Non-fatal error handling: `|| echo "Warning: ..."` prevents set -e from aborting the script. QMD update always runs regardless of gitnexus outcome. |
</threat_model>
