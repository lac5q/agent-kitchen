## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `oklch(0.145 0 0)` — `var(--background)` | Page background |
| Secondary (30%) | `oklch(0.205 0 0)` — `var(--card)` | Card backgrounds, sidebar |
| Accent (10%) | Category colors (sky, emerald, amber, purple, slate) | Category borders, bars, badges — see reserved list |
| Muted text | `oklch(0.556 0 0)` — `var(--muted-foreground)` | Secondary labels, freshness date |
| Destructive | `oklch(0.704 0.191 22.216)` — `var(--destructive)` | Destructive actions only |

**Accent reserved for:**
- Left border of CollectionCard (category-keyed: `border-sky-500`, `border-emerald-500`, `border-amber-500`, `border-purple-500`, `border-slate-500`)
- Fill bar inside CollectionCard (category-keyed bar color)
- Category badge text (`text-sky-400`, `text-emerald-400`, etc.)
- Category text label below bar (`text-*-400` pattern)

**Freshness date color:** `text-slate-500` — uses the existing muted pattern, not a new accent color. Matches `text-muted-foreground` intent.

Source: `globals.css` CSS variables (verified), `collection-card.tsx` CATEGORY_COLORS/CATEGORY_TEXT_COLORS maps (verified)

### Change: CollectionCard freshness date
Add below the category text line (after line 66 in current `collection-card.tsx`):

```tsx
{collection.lastUpdated && (
  <p className="text-xs text-slate-500 mt-0.5" title={collection.lastUpdated}>
    {new Date(collection.lastUpdated).toLocaleDateString()}
  </p>
)}
```

**Exact placement:** Below `<p className={...}>{category}</p>`, still inside `CardContent`.
**Why `text-slate-500`:** Matches existing muted text pattern in dark card context. `var(--muted-foreground)` maps to `oklch(0.556 0 0)` ≈ slate-500.
**Why `title={collection.lastUpdated}`:** Provides ISO timestamp on hover for precision when relative date is ambiguous.

Source: RESEARCH.md Pattern 3 (verified from `KnowledgeCollection` type and `health-panel.tsx` usage pattern)

## Copywriting Contract

| Element | Copy |
|---------|------|
| Freshness label (tooltip) | ISO date string — no user-facing label, raw `toLocaleDateString()` output |
| Empty state (no lastUpdated) | Component renders nothing — `lastUpdated` is nullable, conditional render, no placeholder copy |
| Error state (API fail) | Existing Library page error handling — not changed in Phase 1 |
| Destructive confirmation | None — Phase 1 has no destructive actions |

**Phase 1 CTA:** None. Phase 1 adds no buttons, modals, or user-initiated actions. The freshness date is read-only display.

Source: REQUIREMENTS.md (KNOW-02 says "show freshness date"), RESEARCH.md (nullable `lastUpdated` type confirmed)

## States and Interactions

| State | Trigger | Behavior |
|-------|---------|----------|
| `lastUpdated` present | API returns a date string | Render `<p>` with locale date below category label |
| `lastUpdated` null | Collection never refreshed or API can't determine mtime | Render nothing (no placeholder, no "never" label) |
| Hover on date | User hovers the `<p>` element | Browser native tooltip shows full ISO timestamp via `title` attribute |

No loading spinners, no skeleton states, no interactive states introduced. The CollectionCard is purely presentational.
