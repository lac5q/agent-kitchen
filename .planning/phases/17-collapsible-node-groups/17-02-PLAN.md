<objective>
Build collapse/expand toggle on top of the parentId migration from Plan 17-01. Clicking a group box adds its id to a `collapsedGroups: Set<string>` state; the `nodes` useMemo applies `hidden: true` to every child whose `parentId` is in the set; the `edges` useMemo applies `hidden: true` to every edge whose source or target is a hidden child. Expand reverses it. Collapsed groups render a summary label + aggregate health color. No persistence — page reload always starts expanded.

Purpose: Per FLOW-12, Luis can declutter the Flow diagram by collapsing dormant agent clusters. Per REQUIREMENTS.md scope note, collapse MUST use `hidden: true` (not array removal) to preserve edge state across toggles. Per STATE.md this is the riskiest change — so toggle logic lives in a pure, unit-tested module (`collapse-logic.ts`), and the canvas just wires state → pure function → React Flow node/edge props.

Output: Fully working collapse/expand with aggregate health color on collapsed group boxes, with the toggle logic implemented as pure testable functions, wired into the canvas via minimal useState + useMemo additions.
</objective>

From src/lib/flow/collapse-logic.ts (to be created):
```typescript
import type { Node, Edge } from "@xyflow/react";

/** Health rank used when computing a collapsed group's aggregate color.
 *  Higher rank wins — so any one error > any one active > all idle, etc. */
export type NodeHealth = "error" | "active" | "idle" | "dormant";

/** Apply `hidden: true` to every child whose parentId is in `collapsedGroups`.
 *  Returns a NEW array — does not mutate. Parent group nodes themselves stay visible. */
export function applyCollapseToNodes(
  nodes: Node[],
  collapsedGroups: ReadonlySet<string>
): Node[];

/** Apply `hidden: true` to every edge whose source OR target id is in `hiddenNodeIds`.
 *  Returns a NEW array — does not mutate. */
export function applyCollapseToEdges(
  edges: Edge[],
  hiddenNodeIds: ReadonlySet<string>
): Edge[];

/** Aggregate hex color for a collapsed group, by max-rank of its children's statuses.
 *  Uses the same palette as FlowNode: error=#f43f5e, active=#10b981, idle=#f59e0b, dormant=#64748b. */
export function aggregateHealthColor(
  childStatuses: ReadonlyArray<string>
): string;
```

React Flow v12 rules confirmed for this plan:
- `node.hidden: true` is the blessed way to hide — edges with hidden endpoints stop rendering automatically IF the edge itself is also hidden or React Flow's edge-hiding runs. Per REQUIREMENTS.md scope note we explicitly set edge.hidden as well to avoid any "dangling to 0,0" regression.
- Group (parent) nodes never get `hidden: true` in this plan — they remain visible as the clickable toggle region.
- Clicking inside the group box area should ONLY toggle when the group itself is the click target, not bubble from a child. The existing `onNodeClick(_, node)` handler in the canvas receives the clicked `Node` — we dispatch on `node.type === "groupBoxNode"` (which currently is EARLY-RETURNED — we flip that behavior).
