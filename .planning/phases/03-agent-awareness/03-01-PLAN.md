Create the shell hook script at `~/.claude/hooks/mem0-session-preload.sh`:

1. **Read stdin JSON** using `jq` (preferred over python3 for simplicity — both available per research):
   - Extract `cwd` via `jq -r '.cwd // ""'`
   - Extract `source` via `jq -r '.source // "startup"'`

2. **Gate on source:** If `source` is NOT `"startup"`, exit 0 immediately. Only fresh sessions get preload (not resume/compact/clear).

3. **Derive project name:** `PROJECT=$(basename "$CWD")`. If empty, exit 0.

4. **Search mem0 — TWO calls** (research mandates both):
   - `curl -sf --max-time 3 "http://localhost:3201/memory/search?q=${QUERY}&agent_id=claude&limit=5"`
   - `curl -sf --max-time 3 "http://localhost:3201/memory/search?q=${QUERY}&agent_id=shared&limit=5"`
   - URL-encode the query using `jq -sRr @uri` (avoid python3 dependency for this).
   - Store results in shell variables.

5. **Merge and format** using `jq` to parse both JSON responses:
   - Extract `.results[] | select(.score >= 0.50) | {score, user_id, memory}` from each response
   - Merge, sort by score descending, take top 5
   - If zero results pass threshold, exit 0 silently (no output = no context injection)

6. **Output formatted block** to stdout:
   ```
   ## Memory Preload — {project_name}

   - [{agent_id}] {memory_text}
   - [{agent_id}] {memory_text}
   ...
   ```

7. **Fail-safe:** Wrap ALL curl and jq calls so any failure results in `exit 0` with no output. Use `set +e` (not `set -e`). Every curl uses `--max-time 3` and `-sf` (silent, fail). The script MUST always exit 0.

8. `chmod +x ~/.claude/hooks/mem0-session-preload.sh`

```markdown
If no Memory Preload block appears (mem0 may be unavailable), call `memory_search`
with the project name to retrieve relevant context manually.
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Hook -> mem0 HTTP | Hook sends project name to localhost:3201; response is treated as data |
| mem0 output -> Claude context | Memory text injected into Claude's context window |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | Project name in curl URL | mitigate | URL-encode project name with `jq -sRr @uri` before inserting into curl URL; `basename` strips path traversal |
| T-03-02 | Tampering | mem0 memory content as prompt injection | accept | mem0 is a trusted local service; memories are human/agent-written facts; output wrapped in `## Memory Preload` header framing it as data, not instruction; limited to 5 items |
| T-03-03 | Denial of Service | Hook blocks session start | mitigate | `curl --max-time 3 -sf` on all HTTP calls; `set +e`; unconditional `exit 0`; timeout 8s in settings.json registration |
| T-03-04 | Information Disclosure | Memory content leaked | accept | All data stays local (localhost:3201 -> stdout -> Claude context); no network egress; no logging |
</threat_model>
