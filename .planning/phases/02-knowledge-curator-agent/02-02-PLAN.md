```
#!/bin/bash
# Nightly Knowledge Curator — orchestrates the full knowledge loop
```

- `export PATH="/opt/homebrew/bin:$PATH"` — ensures qmd and npx are found in cron context

**Env loading (CRITICAL — Pitfall 1):**
- `set -a && source ~/github/knowledge/.env && set +a`
- This MUST come before any Python script calls so QDRANT_API_KEY and GEMINI_API_KEY are exported

**Timestamp prefix function:**
- `log() { echo "[$(date '+%Y-%m-%d %H:%M:%S')] $*"; }`

**Do NOT use set -e** — each step must be non-fatal. Use `|| log "Warning: step N failed (non-fatal)"` pattern instead.

**Four steps in sequence:**

Step 1 — GitNexus analyze:
```bash
log "[1/4] GitNexus analyze..."
~/github/gitnexus-index.sh || log "  Warning: gitnexus-index failed (non-fatal)"
```

```bash

~/github/knowledge/llm-wiki-process.sh || log "  Warning: llm-wiki check failed (non-fatal)"
```

Step 3 — mem0 export:
```bash
log "[3/4] mem0 highlights export..."
~/github/knowledge/mem0-export.sh || log "  Warning: mem0 export failed (non-fatal)"
```

Step 4 — QMD update (BM25) + Qdrant index (vectors):
```bash
log "[4/4] QMD update + Qdrant indexing..."
qmd update || log "  Warning: qmd update failed (non-fatal)"
~/github/knowledge/.venv/bin/python3 ~/github/knowledge/qdrant-indexer.py || log "  Warning: Qdrant indexer failed (non-fatal)"
```

**Footer:**
```bash
log "Knowledge Curator complete."
```

chmod +x the script. Validate with `bash -n`.
