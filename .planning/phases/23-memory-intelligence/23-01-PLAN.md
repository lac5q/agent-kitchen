CREATE TABLE IF NOT EXISTS memory_salience (
  message_id     INTEGER PRIMARY KEY REFERENCES messages(id) ON DELETE CASCADE,

**src/lib/memory-consolidation.ts:**

Export `runConsolidation()` and `startConsolidationScheduler()`.

`runConsolidation()`:
1. Check `process.env.ANTHROPIC_API_KEY` -- if missing, `console.warn('[consolidation] ANTHROPIC_API_KEY not set -- consolidation disabled')` and return.
2. `const db = getDb()`
3. Create a run row: `INSERT INTO memory_consolidation_runs(batch_size) VALUES(0)` -- get `lastInsertRowid`.
4. Select batch: `SELECT id, content FROM messages WHERE consolidated = 0 LIMIT 50`.
5. If empty batch, update run to `status='completed', batch_size=0, insights_written=0, completed_at=now`. Return.
6. Send batch to Anthropic: `new Anthropic().messages.create({ model: 'claude-haiku-4-5', max_tokens: 1024, messages: [{ role: 'user', content: prompt }] })`. The prompt asks LLM to analyze memory fragments and return a JSON array of `[{"insight_type": "pattern"|"contradiction"|"summary", "content": "description"}]`.
7. Strip markdown code fences from response text (replace triple-backtick blocks), JSON.parse in try/catch. On failure, log and set insights to empty array.
8. For each parsed insight, INSERT into `memory_meta_insights(run_id, insight_type, content, source_ids)` where `source_ids` = JSON.stringify of batch message IDs array.
9. Mark batch messages: `UPDATE messages SET consolidated = 1 WHERE id IN (...)` using parameterized placeholders.
10. Update run: `status='completed', batch_size=N, insights_written=M, completed_at=strftime(...)`.
11. Wrap entire function body in try/catch -- on error, update run `status='failed', error_message=err.message`.

`startConsolidationScheduler()`:
- Module-level `let _started = false` guard.
- If `_started`, return immediately.
- Set `_started = true`.
- Log `[consolidation] scheduler started (interval: 15m)`.
- Call `runConsolidation()` immediately (fire-and-forget, catch errors).
- `setInterval(() => { runConsolidation().catch(console.error) }, 15 * 60 * 1000)`.

**src/lib/memory-decay.ts:**

Module-level LOG() probe (runs once on first import):
```typescript
import { getDb } from './db';

let _hasLogFn: boolean | null = null;

function hasLogFunction(): boolean {
  if (_hasLogFn === null) {
    try {
      getDb().prepare('SELECT LOG(1.0)').get();
      _hasLogFn = true;
    } catch {
      _hasLogFn = false;
    }
  }
  return _hasLogFn;
}
```

Expose `_resetForTest()` that sets `_hasLogFn = null` for test isolation.

Decay rates map:
```typescript
const DECAY_RATES: Record<string, number> = {
  high: 0.01,
  mid: 0.02,
  low: 0.05,
};
// pinned is NOT in the map -- it is never processed
```

`runDecay()`:
1. `const db = getDb()`
2. For each tier in ['high', 'mid', 'low'] (skip pinned):
   - If `hasLogFunction()`: run access-resistance SQL:
     ```sql
     UPDATE memory_salience
     SET salience_score = MAX(0.0, salience_score * (1.0 - (? / (1.0 + LOG(1.0 + CAST(access_count AS REAL)))))),
         last_decay_at = strftime('%Y-%m-%dT%H:%M:%SZ','now')
     WHERE tier = ? AND date(last_decay_at) < date('now')
     ```
   - Else: flat-rate SQL:
     ```sql
     UPDATE memory_salience
     SET salience_score = MAX(0.0, salience_score * (1.0 - ?)),
         last_decay_at = strftime('%Y-%m-%dT%H:%M:%SZ','now')
     WHERE tier = ? AND date(last_decay_at) < date('now')
     ```
   - Bind parameters: `(rate, tier)`.
3. Log `[decay] cycle complete`.

`startDecayScheduler()`:
- Same `_started` guard pattern as consolidation.
- Log `[decay] scheduler started (interval: 60m)`.
- Call `runDecay()` immediately, then `setInterval(runDecay, 60 * 60 * 1000)`.

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-01 | Injection | /api/agent-peers window param | mitigate | Parse as Number, clamp to [1, 1440], bind as parameter to datetime() -- no string interpolation in SQL |
| T-23-02 | Information Disclosure | /api/memory-stats | accept | Read-only stats endpoint; no PII in tier counts or run metadata; single-user local tool |
| T-23-03 | Tampering | POST /api/memory-consolidate | mitigate | No user input in consolidation query; batch is SELECT from own DB; LLM output parsed in try/catch, never executed as code |
| T-23-04 | Information Disclosure | consolidation -> Anthropic API | accept | Memory content sent to Anthropic API for analysis; this is the intended purpose; user must provide own API key |
| T-23-05 | Denial of Service | consolidation scheduler | mitigate | LIMIT 50 per batch; _started guard prevents duplicate intervals; try/catch prevents crash-loop |
| T-23-06 | Tampering | LLM response parsing | mitigate | Strip code fences, JSON.parse in try/catch, validate insight_type against allowlist before INSERT; never eval() |
