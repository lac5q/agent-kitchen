Add the following to ~/github/knowledge/personal-ingestion-transcripts.py:

New imports (add at top if not already present): import sqlite3

New constant:
SPARK_DB = Path("/Users/yourname/Library/Application Support/Spark Desktop/core-data/messages.sqlite")
SPARK_DIR = KNOWLEDGE_ROOT / "spark-recordings"

New function: ingest_spark(state: dict) -> tuple[dict, str]
1. Verify SPARK_DB exists: if not SPARK_DB.exists(): print("Warning: Spark DB not found"); return state, ""
2. Connect read-only: conn = sqlite3.connect(f"file:{SPARK_DB}?mode=ro", uri=True)
3. Set conn.row_factory = sqlite3.Row
4. Query:
   SELECT mte.rowid, mte.summary, mte.startDate,
          m.subject, m.messageFrom, m.messageTo
   FROM meetTranscriptEvent mte
   JOIN messages m ON m.pk = mte.messagePk
   WHERE mte.rowid > ?
   ORDER BY mte.startDate ASC
   Pass state["spark"]["last_message_rowid"] as parameter.
5. SPARK_DIR.mkdir(parents=True, exist_ok=True)
6. meetings_snippets = [] (collect text for daily note Meetings section)
7. For each row:
   a. Convert startDate (unix seconds) to datetime: dt = datetime.datetime.fromtimestamp(row["startDate"])
   b. date_str = dt.strftime("%Y-%m-%d")
   c. Title: use row["subject"] if available, else row["summary"]
   d. Attendees: parse row["messageTo"] (comma-separated emails)
   e. Content: row["summary"] is short (26-39 chars -- this is a title, not full transcript per research). Use as meeting title/topic.
   f. Write markdown to SPARK_DIR / "{date_str}-{rowid}.md" with frontmatter:
      ---
      rowid: {rowid}
      title: {title}
      date: {date_str}
      attendees: [{attendees}]
      source: spark_desktop
      ---
      # {title}
      **Date:** {date_str} {dt.strftime("%H:%M")}
      **Attendees:** {attendees}
      **Summary:** {row["summary"]}
   g. Add mem0 memory: add_mem0("Spark meeting: {title} on {date_str} with {attendees}. Topic: {summary}", "luis", {"type": "spark_transcript", "rowid": rowid, "date": date_str})
   h. **Obsidian meeting note (D-07):** Infer project_name from title/summary (simple heuristic: look for known project keywords or attendee email domains; fall back to "general"). Call create_project_meeting_note(date_str, title, attendees, row["summary"], project_name) to write the project-based meeting note file.
   i. Collect snippet for daily note: meetings_snippets.append(f"### {title} ({date_str})\nAttendees: {attendees}\n{row['summary']}")
   j. Track max rowid for state update
8. Close connection
9. Update state["spark"]["last_message_rowid"] to max rowid seen
10. Print: "Ingested {N} Spark transcript events"
11. Return updated state AND "\n\n".join(meetings_snippets) as meetings_text

Update ingest_transcripts() main function -- after the existing Drive ingestion and meetings section logic:
  state, spark_meetings_text = ingest_spark(state)
  If spark_meetings_text is non-empty, fold it into the daily note Meetings section:
    append_daily_note(today, "Meetings", spark_meetings_text)
  (This appends Spark meetings alongside any Drive meeting notes already in the Meetings section.)

Also update state["spark"]["last_run"] to now before save_state(state).
