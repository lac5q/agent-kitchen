```python
GWS = "/Users/lcalderon/bin/gws"
EMAILS_DIR = Path.home() / "github/knowledge/emails"
MAX_THREADS_PER_RUN = 50  # Safety cap per 6h run
RATE_LIMIT_MS = 100  # 100ms between thread fetches
```

**Function: `gws_cmd(args: list) -> dict`**
Run gws CLI via `subprocess.run([GWS] + args, capture_output=True, text=True, timeout=30)`. Parse stdout as JSON. Return parsed dict. On failure, print warning and return `{}`.

**Function: `list_threads(after_date: str) -> list[dict]`**
Call: `gws gmail users threads list --params '{"userId": "me", "q": "from:me after:{after_date}", "maxResults": 500}' --page-all`
Parse result — gws returns `{"threads": [...]}` with each thread having `id` and `snippet`. Return list of thread dicts. If no threads field, return `[]`.

**Function: `fetch_thread(thread_id: str) -> dict`**
Call: `gws gmail users threads get --params '{"userId": "me", "id": "{thread_id}", "format": "full"}'`
Return full thread object.

**Function: `thread_to_markdown(thread: dict) -> str`**
Extract from thread:
- Subject from first message headers (find header with name "Subject")
- Date from first message headers (find header with name "Date")
- From/To from each message headers
- Body from each message using `get_body(message["payload"])`

Format as markdown:
```
~~~yaml-frontmatter-start~~~
thread_id: {id}
subject: {subject}
date: {date}
participants: [{from_addresses}]
~~~yaml-frontmatter-end~~~

# {subject}

## Message 1 — {from} ({date})

{body}

## Message 2 — {from} ({date})

{body}
...
```

**Function: `build_digest_entry(subject: str, msg_count: int, snippet: str) -> str`**
Return: `- **{subject}** ({msg_count} messages) -- {snippet[:100]}`

**Function: `ingest_emails() -> None`**
1. `ensure_dirs()`
2. `state = load_state()`
3. Convert `state["gmail"]["last_run"]` to Gmail date format `YYYY/MM/DD` for the `after:` query
4. `threads = list_threads(after_date)`
5. Filter out threads whose `id` is in `state["gmail"]["ingested_thread_ids"]`
6. Cap at `MAX_THREADS_PER_RUN`
7. For each thread:
   a. `full_thread = fetch_thread(thread["id"])`
   b. `md = thread_to_markdown(full_thread)`
   c. Extract date from first message for filename: `YYYY-MM-DD-{thread_id}.md`
   d. Write to `EMAILS_DIR / filename`
   e. Build digest entry
   f. Append thread_id to `state["gmail"]["ingested_thread_ids"]`
   g. `time.sleep(RATE_LIMIT_MS / 1000)`
8. If any threads processed:
   a. Build digest content (all bullet entries joined by newline)
   b. Call `append_daily_note(today_str, "Email Digest", digest_content)`
9. Update `state["gmail"]["last_run"]` to current ISO8601
10. `save_state(state)`
11. Print summary: `"Ingested {N} email threads, {skipped} skipped (already ingested)"`

**Main block:**
```python
if __name__ == "__main__":
    ingest_emails()
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| gws CLI → Gmail API | OAuth-authenticated, reads user's own email |
| Email body content → markdown file | Untrusted email content written to filesystem |
| Email body → Obsidian daily note | Untrusted content appended to personal notes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | I (Info Disclosure) | Email body in markdown | accept | Files stored in local gitignored knowledge repo, same security as email client itself |
| T-05-06 | T (Tampering) | Malicious email body in markdown | mitigate | HTML stripped before writing; no script execution context in markdown files; content is data only |
| T-05-07 | D (DoS) | Unbounded thread fetch | mitigate | MAX_THREADS_PER_RUN=50 cap + 100ms rate limit between fetches prevents runaway API usage |
| T-05-08 | T (Tampering) | Path traversal in email subject→filename | mitigate | Filename uses thread_id (alphanumeric) not subject; slugify() strips all non-alnum chars |
</threat_model>
